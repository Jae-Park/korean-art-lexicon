Standards & Interoperability · 표준과 상호운용성

Korean Art Lexicon is built on international standards to ensure interoperability with global art information systems.

Korean Art Lexicon은 국제 표준을 기반으로 구축되어 글로벌 미술 정보 시스템과의 상호운용성을 보장합니다.

Authority Files 권위 파일

  • Getty AAT — Art & Architecture Thesaurus. Used for classifying entity types (exhibition, organization, term, publication) and roles (artist, curator, critic). 미술·건축 시소러스. 전시 유형, 기관 유형, 용어 분류, 역할 등 모든 타입/분류 체계의 기준으로 사용합니다.
  • Getty ULAN — Union List of Artist Names. Links persons and organizations to the Getty's artist authority records. 통합 작가명 목록. 인물과 기관을 Getty의 작가 전거 레코드에 연결합니다.
  • VIAF — Virtual International Authority File. Clusters authority records from national libraries worldwide, including Korea's National Library (KRNLK). 가상 국제 전거 파일. 한국국립중앙도서관(KRNLK)을 포함한 전 세계 국립도서관의 전거 레코드를 연결합니다.
  • Wikidata — Open knowledge base. Q-item IDs connect lexicon entries to the broader linked data ecosystem. 개방형 지식 베이스. Q-item ID로 렉시콘 항목을 광범위한 링크드 데이터 생태계에 연결합니다.
  • AAA — Asia Art Archive. Connects to the primary archive for contemporary Asian art documentation. 아시아 아트 아카이브. 아시아 현대미술 기록의 핵심 아카이브와 연결합니다.

ISO Standards ISO 표준

  • ISO 639-1 — Language codes: ko en ja zh 언어 코드
  • ISO 15924 — Script codes: Hang (한글) Hani (한자) Kore (한국어 혼용) Latn (로마자) Jpan (일본어) 문자 코드
  • ISO 3166-1 — Country codes: KR US JP etc. 국가 코드
  • ISO 8601 — Date format: YYYY-MM-DD 날짜 형식

Name Variant Recording 이름 변형 기록

Each entity records name/title variants with structured metadata, documenting how names actually appear across different sources and contexts.

각 항목은 이름/제목 변형을 구조화된 메타데이터와 함께 기록하여, 다양한 출처와 맥락에서 이름이 실제로 어떻게 나타나는지를 문서화합니다.

variants:
  - form: "Lee U-Fan"
    lang: en          # ISO 639-1
    script: Latn      # ISO 15924
    type: alternate
    source: "Tate"
    accessed: 2026-03-30

Interoperability 상호운용성

  • Each entry can link to up to 4 authority files: Getty ULAN, VIAF, Wikidata, and AAA 각 항목은 최대 4개의 권위 파일에 연결됩니다: Getty ULAN, VIAF, Wikidata, AAA
  • Getty AAT IDs enable semantic interoperability — any system using AAT can interpret our type classifications Getty AAT ID를 통해 의미론적 상호운용이 가능합니다 — AAT를 사용하는 모든 시스템이 우리의 분류를 해석할 수 있습니다
  • VIAF links connect to 50+ national library systems, enabling discovery through WorldCat and library OPACs VIAF 연결을 통해 50개 이상의 국립도서관 시스템과 연결되며, WorldCat 및 도서관 OPAC에서의 탐색이 가능합니다
  • Wikidata Q-IDs bridge to Wikipedia, Google Knowledge Graph, and thousands of other linked data consumers Wikidata Q-ID를 통해 위키피디아, Google Knowledge Graph 및 수천 개의 링크드 데이터 소비자에 연결됩니다
  • The full dataset will be published as a public Git repository with structured YAML files and a JSON API endpoint 전체 데이터셋은 구조화된 YAML 파일과 JSON API 엔드포인트를 갖춘 공개 Git 리포지토리로 공개될 예정입니다

Technology 기술 스택

  • Data format — YAML (human-readable, Git-friendly) 데이터 형식 — YAML (사람이 읽을 수 있고, Git 친화적)
  • Validation — JSON Schema + automated URL verification pipeline 검증 — JSON 스키마 + 자동 URL 검증 파이프라인
  • Build — YAML → JSON (static build, no server required) 빌드 — YAML → JSON (정적 빌드, 서버 불필요)
  • Frontend — Vanilla HTML/CSS/JS (no framework dependencies) 프론트엔드 — 순수 HTML/CSS/JS (프레임워크 의존성 없음)
  • Hosting — GitHub Pages (static, free, version-controlled) 호스팅 — GitHub Pages (정적, 무료, 버전 관리)
