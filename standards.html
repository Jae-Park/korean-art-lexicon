Standards & Interoperability · 표준과 상호운용성

Korean Art Lexicon is built on international standards to ensure interoperability with global art information systems.

Korean Art Lexicon은 국제 표준에 기반하여, 전 세계 미술 정보 시스템과 연동할 수 있도록 설계되었습니다.

Authority Files 전거 파일

  • Getty AAT — Art & Architecture Thesaurus. Used for classifying entity types (exhibition, organization, term, publication) and roles (artist, curator, critic). 미술·건축 분류체계(Art & Architecture Thesaurus). 전시 유형, 기관 유형, 용어 분류, 역할 등 모든 분류의 기준으로 사용합니다.
  • Getty ULAN — Union List of Artist Names. Links persons and organizations to the Getty's artist authority records. 통합 작가명 목록(Union List of Artist Names). 인물과 기관을 Getty의 전거 레코드에 연결합니다.
  • VIAF — Virtual International Authority File. Clusters authority records from national libraries worldwide, including Korea's National Library (KRNLK). 가상 국제 전거 파일(Virtual International Authority File). 한국국립중앙도서관(KRNLK)을 포함, 전 세계 국립도서관의 전거 레코드를 클러스터로 연결합니다.
  • Wikidata — Open knowledge base. Q-item IDs connect lexicon entries to the broader linked data ecosystem. 개방형 지식 베이스. Q-item ID를 통해 렉시콘 항목을 링크드 데이터 생태계에 연결합니다.
  • AAA — Asia Art Archive. Connects to the primary archive for contemporary Asian art documentation. 아시아 아트 아카이브(Asia Art Archive). 아시아 현대미술 기록의 주요 아카이브입니다.

ISO Standards ISO 표준

  • ISO 639-1 — Language codes: ko en ja zh 언어 코드
  • ISO 15924 — Script codes: Hang (한글) Hani (한자) Kore (한국어 혼용) Latn (로마자) Jpan (일본어) 문자 코드
  • ISO 3166-1 — Country codes: KR US JP etc. 국가 코드
  • ISO 8601 — Date format: YYYY-MM-DD 날짜 형식

Name Variant Recording 이름 변형 기록

Each entity records name/title variants with structured metadata, documenting how names actually appear across different sources and contexts.

각 항목의 이름/제목 변형을 구조화된 메타데이터와 함께 기록합니다. 출처별로 실제 사용된 표기를 추적합니다.

variants:
  - form: "Lee U-Fan"
    lang: en          # ISO 639-1
    script: Latn      # ISO 15924
    type: alternate
    source: "Tate"
    accessed: 2026-03-30

Interoperability 상호운용성

  • Each entry can link to up to 4 authority files: Getty ULAN, VIAF, Wikidata, and AAA. 각 항목은 최대 4개의 전거 파일과 연결할 수 있습니다: Getty ULAN, VIAF, Wikidata, AAA
  • Getty AAT IDs enable semantic interoperability — any system using AAT can interpret our type classifications. Getty AAT ID 기반 분류를 사용하므로, AAT를 채택한 모든 시스템과 의미론적으로 호환됩니다
  • VIAF links connect to 50+ national library systems, enabling discovery through WorldCat and library OPACs. VIAF를 통해 50개 이상의 국립도서관 시스템과 연결되며, WorldCat 및 도서관 목록에서 탐색할 수 있습니다
  • Wikidata Q-IDs bridge to Wikipedia, Google Knowledge Graph, and thousands of other linked data consumers. Wikidata Q-ID를 통해 위키피디아, Google 지식 패널 등 다양한 링크드 데이터 시스템과 연결됩니다
  • The full dataset will be published as a public Git repository with structured YAML files and a JSON API endpoint. 전체 데이터셋은 구조화된 YAML 파일과 JSON API 엔드포인트를 갖춘 공개 Git 리포지토리로 제공할 예정입니다

Technology 기술 스택

  • Data format — YAML (human-readable, Git-friendly) 데이터 형식 — YAML (사람이 읽을 수 있는 형식, Git과 호환)
  • Validation — JSON Schema + automated URL verification pipeline 검증 — JSON 스키마 + 자동 URL 검증 파이프라인
  • Build — YAML → JSON (static build, no server required) 빌드 — YAML → JSON (정적 빌드, 서버 불필요)
  • Frontend — Vanilla HTML/CSS/JS (no framework dependencies) 프론트엔드 — 순수 HTML/CSS/JS (프레임워크 의존성 없음)
  • Hosting — GitHub Pages (static, free, version-controlled) 호스팅 — GitHub Pages (정적, 무료, 버전 관리)
  • License — Data: CC-BY-4.0 · Code: MIT 라이선스 — 데이터: CC-BY-4.0 · 코드: MIT

As the lexicon scales beyond 1,000 entries, the technology stack may transition to a database-backed architecture (PostgreSQL) with user authentication and a public API, while maintaining the same data standards and authority file integrations described above.

렉시콘이 1,000개 항목을 넘어 확장될 경우, 사용자 인증과 공개 API를 갖춘 데이터베이스 기반 아키텍처(PostgreSQL)로 전환될 수 있습니다. 위에 기술된 데이터 표준과 전거 파일 연동은 그대로 유지됩니다.
