Seoul Reading Room

Korean Art Lexicon is designed and developed by Jaeyong Park—translator, writer, and curator—as an initiative of Seoul Reading Room. This project aims to serve the Korean art translation community as a public resource.

Korean Art Lexicon은 통번역가, 필자, 큐레이터 박재용이 기획·개발하고, 서울리딩룸의 이니셔티브로 운영됩니다. 한국미술 번역 커뮤니티를 위한 공공 자원을 목표로 합니다.

Currently in prototype development. · 현재 프로토타입 개발 중입니다.

Why

왜 필요한가

Korean art names lack a shared standard. The same artist appears as three different people across databases. Exhibition titles change between press release and catalogue.

한국미술의 이름에는 공유된 기준이 없습니다. 같은 작가가 데이터베이스마다 다른 사람으로 등장하고, 전시 제목은 보도자료와 도록 사이에서 달라집니다.

We document how names are actually used—not how they should be used. Every entry traces its source. Every variation is recorded with evidence.

우리는 이름이 실제로 어떻게 사용되고 있는지를 기록합니다. 모든 항목은 출처를 추적하고, 모든 표기 변형은 근거와 함께 기록됩니다.

Open data. Open contributions. Open governance.

열린 데이터. 열린 기여. 열린 거버넌스.

Scope 범위

  • Persons — Artists, curators, writers, critics 인물 — 작가, 큐레이터, 필자, 비평가
  • Exhibitions — Titles in Korean and English 전시 — 한국어·영어 전시 제목
  • Organizations — Museums, galleries, artist-run spaces 기관 — 미술관, 갤러리, 대안공간
  • Terms — Dansaekhwa, Minjung art, Sinseang gonggan... 용어 — 단색화, 민중미술, 신생공간...
  • Publications — Exhibition catalogues, monographs, periodicals 출판물 — 전시 도록, 단행본, 정기간행물

Principles 원칙

  • Every entry requires at least one verifiable source 모든 항목은 검증 가능한 출처가 필요합니다
  • Preferred forms are distinguished from variants 선호 표기와 변형 표기를 구분합니다
  • Sources track which name form they use 출처별로 사용된 표기를 추적합니다
  • Classifications follow Getty AAT — no custom taxonomies 분류 체계는 Getty AAT를 따릅니다 — 자체 분류를 만들지 않습니다
  • All changes are versioned through Git — the full dataset will be published as a public repository 모든 변경은 Git으로 관리되며, 향후 전체 데이터셋을 공개 리포지토리로 제공할 예정입니다

Roadmap 로드맵

  • Near-term — Expand to 100+ entries: GOKAMS terminology, Gwangju Biennale, major publications. Systematic VIAF and Wikidata registration. 단기 — 항목 100개 이상: GOKAMS 용어 확장, 광주비엔날레, 주요 출판물 추가. VIAF·Wikidata 체계적 등록.
  • Mid-term — 100–500 entries: additional institutional data (ARKO, SeMA, MoCA Busan), foreign artist Korean names, foreign exhibition Korean titles, Wikidata and Getty ULAN contributions. 중기 — 항목 100–500개: 추가 기관 데이터(아르코, 서울시립미술관, 부산현대미술관), 외국 작가 한국어명, 외국 전시 한국어 제목, Wikidata·Getty ULAN 기여.
  • Long-term — 500+ entries: transition to database-backed architecture (PostgreSQL) with user registration and permission tiers (viewer → contributor → reviewer → editor → admin), dedicated domain, public API endpoint, open dataset as a public Git repository. 장기 — 항목 500개 이상: 데이터베이스 기반 아키텍처(PostgreSQL)로 전환. 사용자 등록 및 권한 계층(열람자 → 기여자 → 리뷰어 → 편집자 → 관리자), 자체 도메인, 공개 API 엔드포인트, 공개 Git 리포지토리로 데이터셋 제공.
